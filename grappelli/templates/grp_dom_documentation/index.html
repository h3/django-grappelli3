The basic structure of a page

Doctype

Grappelli is based on HTML 5 therefor doctype always needs to be ...


<!DOCTYPE html>

Head

The head of the page looks like this ...


<head>
    <title>{% block title %}{% endblock %}</title>
    <meta charset="utf-8">
    {% block blockbots %}
        <meta name="robots" content="NONE,NOARCHIVE" />
    {% endblock %}
    {% block stylesheets %}
        <link href="{% admin_media_prefix %}stylesheets/screen.css" rel="stylesheet" type="text/css" />
        {% if LANGUAGE_BIDI %}<link href="{% block stylesheet_rtl %}{% admin_media_prefix %}stylesheets/rtl.css{% endblock %}" rel="stylesheet" type="text/css" />{% endif %}
    {% endblock %}
    {% block extrastyle %}{% endblock %}
    {% block javascripts %}
        The js go in here
    {% endblock %}
    {% block extrahead %}{% endblock %}
</head>

Body

The body of the page basically contains three elements:

  1. header <header id="grp-header" />
  2. article <article id="grp-content" />
  3. footer <footer id="grp-footer" />

<body>
    <header id="grp-header">
        Admin-Title, User-Tools, Breadcrumbs etc.
    </header>
    <article id="grp-content">
        The content
    </article>
    <footer id="grp-footer">
        Actions (Save, Delete etc.)
    </footer>
</body>

Body > header <header id="grp-header" />

The header itself contains the following elements:

  1. an include for the actual header {% include "admin/includes_grappelli/header.html" %}
  2. the breadcrumbs <nav id="grp-breadcrumbs" /> which contain a {% block breadcrumbs %} themselves to let you define your page-specific breadcrumbs

<header id="grp-header">
    {% include "admin/includes_grappelli/header.html" %}
    <nav id="grp-breadcrumbs">
        {% block breadcrumbs %}
            <ul>
                <li><a href="/">{% trans 'Home' %}</a></li>
                {% if title %}
                    <li> &rsaquo; {{ title }}</li>
                {% endif %}
            </ul>
        {% endblock %}
    </nav>
</header>
